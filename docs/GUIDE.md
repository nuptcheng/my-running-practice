## 快速开始

### 环境准备

1. 确保已安装 Python 3.6+
2. 安装必要的依赖包：

```bash
pip install pandas matplotlib seaborn
```

### 推荐工作流程

1. **查看本月训练计划** → `training-plans/2026-01-plan.md`
2. **按计划进行训练**
3. **跑步后快速记录** → 使用 `python scripts/quick_log.py`
4. **对比计划完成情况** → 自动显示对比结果
5. **更新月度记录文件** → 将记录添加到 `data/2026/01-January.md`
6. **定期查看进度** → README 中的训练计划概览

## 训练工作流程

### 1. 月初：查看训练计划

打开 `training-plans/当前月份-plan.md`，了解：
- 本月训练目标
- 每周训练安排
- 心率和配速要求

### 2. 训练前：确认当天计划

在训练计划中找到今天的安排：
- 计划距离
- 目标心率
- 目标配速

### 3. 训练中：监控数据

- 实时监控心率（手表/手环）
- 控制配速
- 注意身体感受

### 4. 训练后：快速记录

**方法一：使用快速记录工具（推荐）**

```bash
python scripts/quick_log.py
```

工具会：
- 引导你输入数据
- 自动计算配速
- 对比训练计划
- 生成记录行

**方法二：手动记录**

直接在月度文件中添加一行记录。

### 5. 更新文件

将生成的记录添加到：
- `data/2026/01-January.md` - 月度记录
- `training-plans/2026-01-plan.md` - 更新计划完成状态

### 6. 查看进度

在 README 中查看：
- 本月完成进度
- 本周训练计划
- 与目标的对比

## 记录跑步数据

### 文件命名规则

月度文件命名格式：`月份数字-英文月份名.md`

示例：
- `01-January.md`
- `02-February.md`
- `12-December.md`

### 数据记录格式

每月文件包含以下部分：

#### 1. 月度目标设定

```markdown
## 本月目标

- 总跑量：100 公里
- 平均配速：< 6:00 分/公里
- 目标体重：70 kg
- 训练次数：12 次
```

#### 2. 跑步记录表格

| 字段 | 说明 | 格式示例 |
|------|------|----------|
| 日期 | 跑步日期 | 2026-01-13 |
| 距离(km) | 跑步距离 | 5.0 |
| 时长(分钟) | 跑步时长 | 30 |
| 配速(分/km) | 平均配速 | 6:00 |
| 平均心率(bpm) | 平均心率 | 150 |
| 最大心率(bpm) | 最大心率 | 165 |
| 体重(kg) | 当天体重 | 70.0 |
| 场地 | 训练场地 | 跑步机/户外/操场 |
| 感受(1-10) | 主观感受 | 8 |
| 备注 | 其他说明 | 状态不错 |

#### 3. 月度总结

月末填写本月的完成情况、收获与进步、需要改进的地方以及下月计划。

### 记录示例

```markdown
| 日期 | 距离(km) | 时长(分钟) | 配速(分/km) | 平均心率(bpm) | 最大心率(bpm) | 体重(kg) | 场地 | 感受(1-10) | 备注 |
|------|----------|------------|-------------|---------------|---------------|----------|------|------------|------|
| 2026-01-13 | 5.0 | 30 | 6:00 | 150 | 165 | 70.0 | 跑步机 | 8 | 状态不错 |
| 2026-01-15 | 8.0 | 50 | 6:15 | 155 | 170 | 69.8 | 户外 | 7 | 有点累 |
| 2026-01-17 | 10.0 | 58 | 5:48 | 160 | 175 | 69.5 | 操场 | 9 | 突破PB |
```

### 数据填写技巧

1. **及时记录**：跑步后立即记录，避免遗忘
2. **保持一致**：使用统一的格式和单位
3. **详细备注**：记录特殊情况（如身体不适、路线变化等）
4. **诚实记录**：如实记录感受和数据，便于分析

## 数据分析

### 运行分析脚本

```bash
cd my-running-practice
python scripts/analyze.py
```

### 分析报告内容

分析脚本会输出以下统计信息：

#### 1. 总体统计
- 总跑步次数
- 总跑步距离
- 总跑步时长
- 平均每次距离
- 平均每次时长
- 平均心率
- 最高心率

#### 2. 体重统计
- 最新体重
- 最高体重
- 最低体重
- 体重变化

#### 3. 月度统计
- 每月训练次数
- 每月总距离
- 每月平均心率
- 每月平均感受

#### 4. 最近记录
- 显示最近5次跑步的详细信息

## 图表生成

### 运行可视化脚本

```bash
python scripts/visualize.py
```

### 生成的图表

脚本会在 `output/` 目录下生成以下图表：

1. **distance_trend.png** - 跑步距离趋势图
   - 展示每次跑步的距离变化

2. **pace_trend.png** - 配速趋势图
   - 展示配速的进步情况（Y轴反转，越低越好）

3. **heart_rate_trend.png** - 心率趋势图
   - 展示平均心率和最大心率的变化

4. **weight_trend.png** - 体重变化趋势图
   - 追踪体重变化情况

5. **monthly_summary.png** - 月度统计图
   - 包含月度跑量和训练次数的柱状图

6. **feeling_distribution.png** - 感受评分分布图
   - 展示训练感受的分布情况

### 查看图表

图表生成后，可以在 `output/` 目录中查看所有PNG格式的图表文件。

## 常见问题

### Q1: 如何添加新的一年？

在 `data/` 目录下创建新的年份文件夹，如 `data/2027/`。

### Q2: 配速如何计算？

配速 = 跑步时长（分钟）÷ 距离（公里）

例如：30分钟跑5公里，配速 = 30 ÷ 5 = 6:00 分/公里

### Q3: 心率数据从哪里获取？

可以使用运动手表、手环或跑步APP记录的心率数据。如果没有心率数据，可以填写 `-`。

### Q4: 感受评分如何打分？

建议标准：
- 1-3分：状态很差，非常疲劳
- 4-6分：一般状态，有些吃力
- 7-8分：状态良好，比较轻松
- 9-10分：状态极佳，非常轻松

### Q5: 如果某次跑步缺少部分数据怎么办？

缺失的数据可以填写 `-`，分析脚本会自动跳过这些数据。

### Q6: 可以修改历史数据吗？

可以。直接编辑对应月份的Markdown文件，修改后重新运行分析脚本即可。

### Q7: 如何备份数据？

建议使用Git进行版本控制：

```bash
git add .
git commit -m "更新跑步记录"
git push
```

### Q8: 图表中文显示乱码怎么办？

确保系统安装了中文字体。可以修改 `visualize.py` 中的字体设置：

```python
plt.rcParams['font.sans-serif'] = ['你的中文字体名称']
```

### Q9: 如何设定合理的训练目标？

建议遵循以下原则：
- 每周跑量增加不超过10%
- 根据当前水平设定配速目标
- 循序渐进，避免过度训练

### Q10: 如何判断训练效果？

观察以下指标：
- 相同配速下心率降低
- 相同心率下配速提升
- 主观感受评分提高
- 恢复速度加快
